\section{Core ML: Long-Context Sequence Modeling}
\label{sec:coreml}

% ---------------------------------------------------------------------
\subsection{Overview}
Long-context modeling is a central bottleneck in modern sequence models for two
reasons. First, vanilla self-attention scales quadratically, $O(T^2)$, in
sequence length $T$, so both compute and memory blow up as context grows.
Second, standard learned positional embeddings cannot extrapolate: a model
trained on $512$ positions has no representation for position $513$. This
section studies architectural alternatives along three independent axes ---
the \emph{attention mechanism}, the \emph{positional encoding}, and a
\emph{convolutional hybrid} block layout --- evaluating each one at a time on
the WikiText-2 language-modeling benchmark.

The codebase is built around three registries (\texttt{ATTENTION\_REGISTRY},
\texttt{POS\_ENCODING\_REGISTRY}, \texttt{ARCHITECTURE\_REGISTRY}) so that
attention type, positional encoding, and block layout can be swapped by name
without touching the model definition. This makes every variant below a
controlled, one-axis-at-a-time change against the same baseline.

\subsection{Baseline Transformer}
\label{sec:baseline}
The baseline is a standard multi-head-attention decoder following
Vaswani et al.~\cite{vaswani2017} with causal masking, trained at context
length 1024 to establish a reference point. Its core attention computation is
\[
\text{Attention}(Q,K,V)=\operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}+M\right)V,
\qquad
M_{ij}=\begin{cases}0,& j\le i,\\ -\infty,& j>i,\end{cases}
\]
which materializes the full $T\times T$ score matrix --- the $O(T^2)$ cost the
variants in Section~\ref{sec:attn} attack.

\paragraph{Why these hyperparameters.} $d_{\text{model}}=256$ over $4$ layers is
expressive enough to separate the variants while remaining trainable on a free
GPU; $d_{\text{ff}}=1024$ follows the standard $4\times$ rule; $8$ heads at
$d_k=32$ respect the convention of keeping head dimension $\ge 32$ for stable
attention; pre-LayerNorm is used for training stability. The baseline has
\textbf{\num{28937297}} parameters.

\paragraph{Baseline result.} At context length 1024 the baseline reaches
best validation PPL \textbf{240.29} (val loss 5.482), peak reserved VRAM
\textbf{9664.7\,MB}, and throughput \textbf{15884.5}\,tokens/sec. A model that
guessed uniformly over the $50{,}257$-token vocabulary would have
$\text{PPL}=50{,}257$, so the baseline sits roughly $209\times$ below random,
confirming it is learning the distribution within a short diagnostic budget.

\subsection{Attention Variants}
\label{sec:attn}
I implement and evaluate three replacements for standard self-attention, each
attacking a different cost, and measure all of them across context lengths
512, 1024, and 2048.

\paragraph{Sliding-Window (Local) Attention.} Each token attends only to the
$w$ most recent tokens, $i-w < j \le i$, via a causal local-window mask. This is
a local-connectivity ablation: it tests whether restricting attention to nearby
tokens hurts quality. The implementation is mask-only --- it applies the window
mask to the dense score matrix rather than using a banded kernel --- so it does
not reduce peak memory at this scale.

\paragraph{Multi-Query Attention (MQA).} All $h$ query heads share a single key
head and a single value head~\cite{shazeer2019}, projecting $K,V$ to $d_k$
instead of $d_{\text{model}}$:
\[
\text{KV cache}=2\,n_{kv}\,d_k\,T \text{ floats},\quad n_{kv}=1 \text{ (MQA)}.
\]
This shrinks the KV cache by a factor $h$ at inference. MQA has
\textbf{\num{28478545}} parameters ($\num{458752}$ fewer than the baseline).

\paragraph{Linear (Kernel) Attention.} Softmax is replaced by the positive
feature map $\phi(x)=\operatorname{ELU}(x)+1$~\cite{katharopoulos2020}, so that
\[
\text{Attention}(Q,K,V)\;\approx\;\frac{\phi(Q)\,\big(\phi(K)^\top V\big)}
{\phi(Q)\,\big(\phi(K)^\top \mathbf{1}\big)}.
\]
Computing $\phi(K)^\top V$ first avoids forming the $T\times T$ matrix, giving
$O(Td^2)$ asymptotics. The $+1$ offset keeps $\phi$ strictly positive so the
attention weights stay non-negative. As implemented here the causal mask is
applied to the similarity matrix for correctness, so the long-context memory
benefit of the streaming prefix-sum form is not yet realized.

reserved VRAM, throughput, and per-epoch time for each variant and context
length. Figures~\ref{fig:mem}--\ref{fig:attnppl} plot memory, throughput, and
quality against context length.

\subsection{Positional Encodings}
\label{sec:pos}
Attention is permutation-invariant, so position must be injected explicitly. I
replace the baseline's learned absolute encoding with three alternatives and
compare them both in-distribution (trained and evaluated at 512) and under
length extrapolation (trained at 512, evaluated at 512/1024/2048).

\paragraph{ALiBi.} Adds no positional vector; instead it biases attention logits
by distance~\cite{press2022}:
\[
\alpha_{ij}=\operatorname{softmax}\!\left(\frac{q_i^\top k_j}{\sqrt{d_k}}-m_h\,|i-j|\right),
\qquad m_h=2^{-8h/H}.
\]
The slopes $m_h$ are fixed (not learned), so the recency bias generalizes
analytically to any length --- the train-short-test-long design.

\paragraph{RoPE.} Rotates each query and key by a position-dependent
angle~\cite{su2024}; because both are rotated, the score depends only on the
relative offset:
\[
\langle R_m q,\,R_n k\rangle = q^\top R_{n-m} k,
\qquad \theta_i = 10000^{-2i/d}.
\]
I additionally implement positional interpolation (scaling the rotation angles
by $\text{train\_len}/\text{eval\_len}$) for context extension, used at the
1024 and 2048 evaluations.

\paragraph{Relative bias.} Learns an additive bias indexed by clipped relative
distance~\cite{shaw2018}, so each head learns a separate bias per offset within
a clipped range.

\paragraph{In-distribution quality.} Trained and evaluated at length 512,
Table~\ref{tab:pe} and Figure~\ref{fig:petrain} show ALiBi achieving the lowest
best validation PPL, with RoPE close behind and relative bias well above both.

\subsection{Convolution + Attention Hybrids}
\label{sec:conv}
Self-attention is a global generalist with only a weak built-in preference for
nearby tokens, whereas convolutions are local specialists. The hybrid gives each
block a cheap explicit local path before global mixing, using a \emph{causal,
depthwise-separable} 1-D convolution: a depthwise conv (one $k$-tap filter per
channel) followed by a $1\times1$ pointwise conv. Left-padding by $k-1$ with no
right context keeps it causal, the convolutional analogue of the causal mask.
Its cost is $O(T\,k\,d)$, negligible beside attention.

\paragraph{Designs implemented.} Two layouts are registered:
\begin{itemize}[nosep]
\item \texttt{conv\_prefix}: a conv sublayer is inserted \emph{before}
self-attention inside every block ($x \to {+}\text{conv} \to {+}\text{attn} \to
{+}\text{ffn}$), so attention always works on locally-smoothed features.
Placing the conv first is deliberate --- placing it after would blur
representations attention had just globally mixed.
\item \texttt{interleaved}: whole blocks alternate, with even layers replacing
attention by a conv token-mixer and odd layers keeping attention.
\end{itemize}

\paragraph{Configuration evaluated.} Following the assignment's "combine your
best choices" instruction, the hybrid couples the standalone sliding-window
attention with ALiBi positional encoding, evaluated at 512/1024/2048. The
reported runs correspond to the \textbf{interleaved} layout (param count
\num{28547153}; the lower @2048 memory below is consistent with half the
attention layers being replaced by conv).%

\subsection{Core ML Results Summary}
\label{sec:coreml-summary}
On \textbf{hardware}, none of the implemented attention variants reduces peak
memory at this scale --- the mask-only sliding window and the dense-mask linear
attention all track the standard baseline (Figure~\ref{fig:mem}) --- so the
honest finding is that realizing sub-quadratic memory requires true banded or
streaming kernels, which are the natural next step. On \textbf{position}, the
two questions have two answers: ALiBi gives the lowest in-distribution PPL
(214.35 at 512) \emph{and} extrapolates flattest (Table~\ref{tab:extrap}), while
RoPE is competitive in-distribution but degrades with length. On
\textbf{architecture}, interleaving a cheap local convolution with attention
improves the short-budget objective, with the hybrid reaching the lowest PPL of
any 1024-context run here (234.35).
